## Vulnerability Details

## Summary:
libcurl 8.20.0 ignores the server-declared HTTP Digest `domain` protection space for origin authentication and reuses stored Digest state too broadly on the same easy handle.

After a successful Digest-authenticated request, a later request on the same easy handle can receive a preemptive `Authorization: Digest ...` header even when the new request is outside the `domain` scope declared by the server's original `WWW-Authenticate` challenge.

In my strongest reproduction:

- request 1 receives `WWW-Authenticate: Digest realm="realmA", nonce="nonceA", algorithm=MD5, domain="/first"`
- request 2 on the same easy handle to `/reuse2` sends a preemptive `Authorization: Digest ...`
- a fresh easy handle to `/fresh2` sends no `Authorization` at all
- forcing a fresh TCP connection for request 2 still leaks the Digest header

This is a Digest protection-space violation with concrete confidentiality impact: Digest auth transcripts are disclosed outside the declared scope and can be used for offline password guessing against low-entropy passwords.

Root cause notes:
the preemptive Digest send path in lib/http_digest.c reuses stored Digest state on the easy handle
lib/vauth/digest.c parses and stores fields such as nonce, realm, opaque, qop, and algorithm, but does not parse or enforce the domain directive
as a result, the server-declared Digest protection space is effectively widened to the full origin for later preemptive reuse on the same easy handle

## Impact

In the reproduced flow, a path outside `domain="/first"` receives:
- `username`
- `realm`
- `nonce`
- `uri`
- `response`

from a preemptively forwarded Digest header that should not have been sent there.

This leaked Digest transcript can be used for offline password guessing in realistic low-entropy-password deployments. I verified that the captured transcript in the attached PoC is sufficient to recover the sample password `sekret` from a small dictionary.

This is not an SSRF claim and not a keep-alive reuse artifact. The security claim is limited to preemptive Digest auth disclosure outside the server-declared protection space.
