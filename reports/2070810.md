## Program Summary

> Subject:     Potential NULL dereference in libssh's sftp server
CVE ID#:     CVE-2023-3603

Versions:    master only

Summary:     Processing SFTP server read may cause NULL dereference

Description

Missing allocation check in sftp server processing read requests may
cause NULL dereference on low-memory conditions. The malicious client
can request up to 4GB SFTP reads, causing allocation of up to 4GB buffers,
which is being unchecked for failure.

This will likely crash the authenticated user sftp server's connection
(if implemented as forking as we recommend). For thread-based
servers, this might cause DoS also for legitimate users.

Patch Availability

Patches addressing the issues have been posted to:
https://www.libssh.org/security/
Given this code is not in any released versions, no security release have
been issued.

Credits

Originally reported by Wei Chong Tan.
Patches provided by Jakub Jelen of the libssh team.
